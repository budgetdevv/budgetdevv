### I'm TrumpMcDonaldz, a dev by hobby! 👋

Here's the story - I started out as a Unity developer. Revolted by the fact that I had to host a headless Unity instance for my multiplayer game, I was on a hunt for a physics engine written purely in C#, in hope that I could write my game server in a .NET Core console app

To my disappointment, most engines were either implemented in lower-level languages ( E.x. C++ ), or a thin C# wrapper wrapping native code. Via a YouTube video, I chanced upon BepuPhysics2 ( https://github.com/bepu/bepuphysics2 ), a blazing fast physics engine written in pure C#! 

That made me realize that the language isn't the limiting factor - my knowledge is! I studied the library like a bible ( The author was also really helpful ), and that basically kickstarted my low-level programming journey

**TLDR** -> I enjoy writing performant algorithms, and studying low-level quirks / optimization techniques
